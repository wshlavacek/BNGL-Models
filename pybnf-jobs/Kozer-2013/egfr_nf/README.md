# egfr_nf — EGFR higher-order clustering, network-free NFsim (PyBNF edition-2 job)

A PyBNF edition-2, parameter-fitting job setup. The **network-free NFsim model** is the
PyBioNetFit / BioNetFit-1 **"example 2"** model (`example2_starting_point.bngl`, edited by
B. R. Thomas, W. S. Hlavacek, and E. D. Mitra) — an **unbounded** network-free reformulation of
the EGFR higher-order-clustering model of Kozer et al. **2013**:

> Kozer N, Barua D, Orchard S, Nice EC, Burgess AW, Hlavacek WS, Clayton AHA.
> **"Exploring higher-order EGFR oligomerisation and phosphorylation — a combined experimental
> and theoretical approach."** *Mol BioSyst* 2013; **9**(7):1849–1863.
> PMCID: [PMC3698845](https://pmc.ncbi.nlm.nih.gov/articles/PMC3698845/) ·
> DOI: [10.1039/c3mb70073a](https://doi.org/10.1039/c3mb70073a)

It is fit to the same average-normalized 2013 cluster-density + phospho-EGFR data as the ODE
sibling [`../egfr_ode`](../egfr_ode/) (Figs. 2B/2D/3B/3D). This slug is the **NFsim (network-free)
formulation** of the 2013 clustering problem — the ODE↔NFsim demonstration twin of `egfr_ode` —
attributed to the PyBNF 2019 paper corpus (Mitra et al., *iScience* 2019; 19:1012–1036;
BioNetFit 1 problem 2), on the edition-2 surface.

> **Provenance note (corrected).** This model is *not* the Kozer **2014** published SI. The 2014
> companion (Kozer et al., *Biochemistry* 2014; 53(16):2594–2604,
> [PMC4010257](https://pmc.ncbi.nlm.nih.gov/articles/PMC4010257/); SI `bi500182x_si_001.txt`) is a
> **tetramer-capped ODE model that adds Grb2** — a *different* formulation, not reproduced here. The
> unbounded network-free model here originates in the 2019 PyBioNetFit "example 2", not the 2014 SI
> file. See [`VALIDATION.md`](VALIDATION.md) Gate 2.

## The model

Same physics as [`../egfr_ode`](../egfr_ode/) — EGF binding, EGFR ectodomain crosslinking,
tail conformational activation and *trans*-phosphorylation — but with **no cap on oligomer
size**: EGFR forms unbounded higher-order oligomers (tracked up to 20-mers), so the reaction
network is not finite and the model is simulated **network-free with NFsim** (`method: nf`).
Molecules-per-cell units (the Faeder et al. 2009 / molecule-count convention). Stochastic.
The unbounded oligomers are expected to reproduce the capped-ODE (`../egfr_ode`) behaviour because
higher-order (>tetramer) oligomers are rare — quantified in [`VALIDATION.md`](VALIDATION.md).

## What is fit

The same four Kozer datasets as `egfr_ode` (each scaled so its measurement average is 1):

| dataset | observable | design | figure |
|---|---|---|---|
| EGFR cluster density vs. time (30 nM) | `pre2_time` = `alpha2_pre`·Clusters/f | NF time course, 0–600 s | Fig. 3B |
| phospho-EGFR vs. time (30 nM) | `pre4_time` = `alpha4_pre`·pEGFR/f | NF time course, 0–600 s | Fig. 3D |
| EGFR cluster density vs. EGF dose | `pre1_dose` = `alpha1_pre`·Clusters/f | NF dose-response scan over `LT_nM`, to t_end=600 s | Fig. 2B |
| phospho-EGFR vs. EGF dose | `pre3_dose` = `alpha3_pre`·pEGFR/f | NF dose-response scan over `LT_nM`, to t_end=600 s | Fig. 2D |

`Clusters` is a model **function** summing the 20 EGFR oligomer-size species; `/f` (f=0.01)
converts per-subvolume counts to per-cell. Both `.exp` carry `_SD` columns → **`chi_sq`**.

## ⚠️ Shipped nominals are a STARTING point, not the fit

The source model is `example2_starting_point.bngl` — every parameter comment says the value
"will be replaced by a value generated by GenFit." Those nominals are the **GenFit starting
values**, ~100× below the average-normalized data. The **PyBNF best-fit** (Mitra 2019 Data S1,
RuleHub [`04-egfrnf/fit_ade`](https://github.com/RuleWorld/RuleHub/tree/2019Aug21/Published/Mitra2019/04-egfrnf),
chi_sq = 13.41) is what reproduces the data, and is used for the figure below:

| id | starting nominal | **PyBNF best-fit** | role |
|---|---|---|---|
| `k_o` | 6 | **3.54931** | ligand-mediated activating tail change (/s) |
| `k_c` | 1.6 | **15.8220** | ligand-independent deactivating tail change (/s) |
| `kaf` | 15.4 | **45.0460** | EGFR tail-tail association (/s) |
| `kar` | 8.89 | **0.64255** | EGFR tail-tail dissociation (/s) |
| `chi_r` | 43700 | **1907.32** | ring-closure enhancement factor (nM) |
| `alpha1_pre` | 2.563e-07 | **2.5718e-05** | cluster-density scale, dose (Fig. 2B) |
| `alpha2_pre` | 2.832e-07 | **2.7509e-05** | cluster-density scale, time (Fig. 3B) |
| `alpha3_pre` | 4.496e-07 | **5.0739e-05** | phospho-EGFR scale, dose (Fig. 2D) |
| `alpha4_pre` | 2.8e-07 | **3.2791e-05** | phospho-EGFR scale, time (Fig. 3D) |

The fit is **sloppy/non-identifiable** — the kinetic constants differ wildly across the paper's
four algorithms at nearly equal objective — but the four `alpha*_pre` scale factors are
well-determined by the data (all algorithms land ~90–106× the nominal). See the paper-level
[`../README.md`](../README.md).

## Changes from the source model (documented in `egfr_nf.bngl`)

- **No `begin actions` block** — a network-free model needs no `generate_network`; the two NF
  experiments are synthesized from the conf. (Contrast `../egfr_ode`, which *keeps* its
  `generate_network`+`max_stoich` directive.)
- Fitted constants bind to model ids **by name** (ADR-0034, no `__FREE`). `kaf`/`chi_r` were
  embedded in `(NA*Vo)` unit conversions in the classic model, so the fitted value is a
  standalone id and the conversion moves to a derived per-cell rate (`kaf_pc`/`chi_r_pc`).

## ✅ PEtab-v2-exportable · ⚠️ Heavy (NFsim cluster-scale)

Verified with `scripts/petab_roundtrip.py --job-type ss --method nf` (export → lint clean →
import). NFsim steps event-by-event over ~10³–10⁴ molecules (up to ~10⁶ ligand particles at
100 nM), so a full fit is a cluster job; the reproduction averages a few replicate NFsim runs.

The runnable `egfr_nf.conf` is the single source of truth; the exported problem is committed under
[`petab/`](petab/) as an illustrative byproduct (the same pattern as PyBNF's own
`examples/tutorial/12_petab_roundtrip/petab/`) for consumers who want PEtab v2 directly without
running PyBNF. It is **generated by `make_petab.py`, never hand-written**, so it can't drift from
the conf — re-run that script after any change. Because the dose-response experiment perturbs
`LT_nM`, the bundle includes `conditions.tsv` + `experiments.tsv`: 4 observables (function names),
9 estimated parameters and 24 measurements over 6 conditions / 6 experiments (one per dose of the
0.001–100 nM scan; the 30 nM time-course points attach to the default condition). The model has no
actions block, so the committed `petab/egfr_nf.bngl` is byte-identical to `egfr_nf.bngl`.

## Verification

- **Tier-1** (`scripts/check_conf.py`): edition 2, `job_type=ss` resolves, both experiments
  bind data, `model.stochastic = True` (→ `simulator='nf'`), 9 free params bind by id, no
  `__FREE`. **PASS.**
- **PEtab round-trip** (`--job-type ss --method nf`): export → PEtab v2 lint clean → re-import.
  **PASS.**
- **Reproduction** (`egfr_nf_reproduction.png`, NFsim at the PyBNF best-fit, replicates
  averaged): reproduces the four Kozer datasets to **median 6–13 % relative error** (cluster
  density time 13.5 %, phospho time 5.9 %, cluster density dose 7.5 %, phospho dose 12.3 %);
  summed chi_sq ≈ **14.9** vs. the published **13.4** (the gap is NFsim stochasticity + finite
  replicates). The `max` outliers are the near-zero t=0 / lowest-dose phospho points.

## Run

```bash
export BNGPATH="$HOME/Simulations/BioNetGen-2.9.3"   # folder with BNG2.pl
cd pybnf-jobs/Kozer-2013/egfr_nf
pybnf -c egfr_nf.conf                 # NFsim is cluster-scale: run large fits on a cluster
python make_petab.py                 # regenerate the committed petab/ bundle from the conf
```
