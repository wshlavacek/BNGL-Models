# PEtab.v2 compliance for edition-2 job setups

What "PEtab.v2-compliant" means operationally, and the constraint it puts on the
`.conf` you author. Grounded in the PyBNF source (v1.6.0). Cites `file:line`.

## The rule: compliance is a *verified property of the native conf*

**Do NOT hand-author PEtab TSV tables.** A real-world example ships the native
edition-2 `.conf` + `.bngl` + `.exp` — the same shape as the whole existing corpus
(`examples/real-world/README.md:1,97-99`). "PEtab.v2-compliant" means that native job:

1. **exports** to a PEtab v2 problem — `pybnf.petab.export_job(conf, out/)` writes
   `problem.yaml` + `parameters.tsv` / `observables.tsv` / `measurements.tsv` (+
   `conditions.tsv` / `experiments.tsv` when it perturbs) + a model copy
   (`pybnf/petab/export.py:131-140,204-223`);
2. **lints clean** — `petab.v2.lint.lint_problem(Problem.from_yaml(...))` after
   `register_bngl()` reports no errors (`docs/petab.rst:86-88`;
   `examples/tutorial/12_petab_roundtrip/README.md:44-48`);
3. **re-imports** — `pybnf.petab.import_job(problem.yaml, back/, job_type=…)`
   reconstructs the same free parameters / priors / noise / data (`import_.py:169-201`;
   `docs/petab.rst:66-67`).

The design intent (ADR-0028): the edition-2 conf is "natively shaped like a PEtab v2
problem — which turns export into transcription." Hand-shaped `problem.yaml` + TSVs
appear only in the *tutorial* PEtab lessons, where the tables themselves are the
teaching subject — and even there they are generated by a `regenerate_fixtures.py`,
never hand-written.

**`scripts/petab_roundtrip.py` runs all three steps** — use it as the compliance gate.

Optionally, you *may* commit an exported `petab/` subfolder as an illustrative byproduct
(as `12_petab_roundtrip/petab/` does), but generate it with `export_job`, don't write
it by hand. For the real-world corpus, the runnable conf is the single source of truth.

## The constraint: stay inside the PEtab-exportable subset

`export_job` **raises `NotImplementedError`** (fail-loud) for features PEtab v2 can't
express. Author the conf to avoid them:

| Not exportable | Why | Instead |
|---|---|---|
| `normalization` (`init`/`peak`/`zero`/`unit`) | no PEtab shape (`export.py:798-827`) | encode the reference in the model/observable, or fit the raw signal |
| `cumulative` | PyBNF-native transform (`export.py:779-795`) | pre-difference the data, or keep native-only |
| `neg_bin` noise | removed from PEtab v2 | use `chi_sq`/`sos`/`sod` or Gaussian/Laplace `noise_model` |
| `lognormal` noise | PyBNF log10 vs PEtab natural log | Gaussian on log-transformed data |
| `kl` / `wasserstein` `profile_objective` | column-joint, no PEtab shape | per-point objective |
| `.con` / `.prop` BPSL constraints on `data:` | PyBNF-native (ADR-0028 addendum) | keep the fit data-only; constraints are a separate native job |
| legacy data linkage (`model = X:Y.exp`, `mutant`, `param_scan` dict) | not new-era | use `experiment:` / `condition:` (you're on edition 2 anyway) |
| Antimony (`.ant`) model | export in native language only | use BNGL (`.bngl`) or SBML (`.xml`) |

**Exportable and sufficient for almost every paper fit:** bare-name or arithmetic
(`observableFormula`) observables; `sos` / `chi_sq` / `sod` / `norm_sos` /
`ave_norm_sos` objectives; per-observable Gaussian or Laplace `noise_model`;
dose-response `parameter_scan`; and `preequilibrate:`. The receptor pre-equilibration
shape and a `chi_sq` time course both round-trip and lint (verified).

> The shipped `igf1r.conf` uses `normalization = init` and is therefore **not**
> exportable — a cautionary example. A PEtab-compliant paper fit must not use it.

## The conf → PEtab v2 mapping (for intuition; ADR-0028)

| edition-2 conf | PEtab v2 target |
|---|---|
| `model: m.bngl` | `model_files: {m: {location, language: bngl}}` |
| `*_var` / `parameter:` free params (bind by id) | `parameters.tsv` rows (`estimate`, bounds, `priorDistribution`/`priorParameters`) |
| model-output data columns; `objective`/`noise_model` | `observables.tsv` (`observableFormula` = bare model name; `noiseFormula`/`noiseDistribution`) |
| `observable: id, formula: expr` | `observables.tsv` row with an arithmetic `observableFormula` |
| `condition: c, perturbations: …` | `conditions.tsv` (`conditionId, targetId, targetValue`) |
| `experiment: e, condition: c, data: f.exp` | `experiments.tsv` period rows + `measurements.tsv` rows tagged `experimentId=e` |
| `experiment: …, type: parameter_scan[, t_end: T]` | one Condition+Experiment per dose; measured at `time=inf` (steady state) or finite `t_end` |
| `experiment: …, preequilibrate: p` | two-period Experiment: `time=-inf` under `p`, then `time=0` |

Free params carry priors both ways (`priorDistribution`/`priorParameters` ↔ `*_var`
family). Per-observable noise: `noiseParameters=<id>` ↔ `noise_model <obs> = <family>,
sigma = fit <id>`.

## Commands (all Python — there is no `--petab` CLI flag)

Run with the PyBNF environment's interpreter and `BNGPATH` set:

```python
from pybnf.petab import export_job, import_job
from pybnf.petab.bngl_model import register_bngl
from petab.v2 import Problem
from petab.v2.lint import lint_problem

export_job("<name>.conf", "petab_out/")                 # conf -> PEtab v2
register_bngl()
report = lint_problem(Problem.from_yaml("petab_out/problem.yaml"))
assert not report.has_errors()                          # PEtab-valid
import_job("petab_out/problem.yaml", "back/", job_type="de")   # PEtab v2 -> runnable conf
```

`scripts/petab_roundtrip.py` wraps exactly this (from inside the example folder, since
conf paths are relative). Lint needs the optional `pybnf[petab]` extra; the script
reports clearly if it's missing (exit 3) so you can install it and re-verify.
