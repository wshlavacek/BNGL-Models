# Data provenance — tracing and re-digitizing an `.exp` to its primary source

How to close **Gate 1**: prove every `.exp` value comes from the primary paper. Two cases —
tabular (easy) and figure (digitize). Reuses `curate-model`'s digitization discipline.

## Case A — the data are in a table (paper or SI)

1. Find the table (main text or SI) the job cites.
2. Transcribe the relevant rows/columns and compare to the shipped `.exp`, aligned on the
   independent variable. They should match within rounding / unit conversion.
3. Record: table id, the unit/normalization the paper used, and the max abs/rel difference.
   `scripts/compare_data.py <transcribed.csv> <shipped.exp>` does the numeric diff.

Watch for: unit conversions (nM ↔ M, molecules ↔ concentration), a normalization the paper applied
(relative-to-control, % of max), and `_SD` columns (SD vs. SEM vs. CI — the objective depends on
this).

## Case B — the data are only in a figure → re-digitize from the PDF

Follow `curate-model` §"Reported Simulation Data Verification":

1. **Render the panel at high resolution.** Read the figure page from the parking-garage PDF (the
   Read tool renders PDF pages as images — request the specific page). If the PDF page is low-res,
   note it as a digitization-uncertainty source.
2. **Identify the exact panel and series** the job fits (e.g. "Fig 5B, filled circles, WT").
3. **Calibrate the axes.** Record two known reference points per axis (from tick labels) and the
   scale (linear / log10). Every data point maps pixel → data via that calibration. Log axes are
   the #1 source of digitization error — calibrate in log space.
4. **Extract the points** at the independent-variable values the `.exp` uses (or the figure's own
   sampling). Record any **legend scale factor** or normalization ("normalized to t=0",
   "% of maximum").
5. **Save** the re-digitized data as `dev/papers/<Author-Year>/<slug>_<fig>_digitized.csv` (in the
   untracked garage) — and, as a *derived* artifact, you may commit a copy under the job folder if
   it becomes the fit target.
6. **Diff** against the shipped `.exp`: `scripts/compare_data.py <digitized.csv> <shipped.exp>`.

## What counts as a PASS

State a tolerance and justify it from the figure resolution, marker size, axis scaling, and
digitization method. A few-percent median difference on a clean linear panel is a PASS; a log-axis
panel read from a rasterized PDF may only support ~10–20 %, which is still a PASS if stated. A
*systematic* offset (all points shifted, or the curve shape wrong) is a FAIL — it usually means the
wrong panel/series, a missed normalization, or a units error.

## Record in VALIDATION.md (Gate 1 row per `.exp`)

`<exp file>` · source = `<paper> Table N / Fig Np, series` · method = transcribed / digitized ·
calibration = `<axes, scale>` · normalization = `<none / to-t0 / %max / …>` · diff = `median X%,
max Y%` · verdict.

## No source for the data?

If the values match no primary table or figure, Gate 1 is **FAIL** — the job fits numbers of
unknown origin. The one legitimate exception is a **synthetic-data** job (e.g. Gupta fceri), where
the `.exp` is generated by a ground-truth model *on purpose*; then Gate 1 = confirm the generator
(ground-truth params) is the paper's, and Gate 3 shifts to ground-truth recovery.
